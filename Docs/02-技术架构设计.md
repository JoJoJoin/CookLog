# 02 · 技术架构设计

## 1. 技术栈总览

| 层 | 选型 | 选择理由 |
| --- | --- | --- |
| 跨平台框架 | **Flutter (stable, Dart 3)** | 一套代码多端运行；先 Android，后续 iOS/桌面/Web 改动最小；UI 表现力强 |
| 状态管理 | **Riverpod (flutter_riverpod)** | 编译期安全、易测试、无需 BuildContext、解耦良好 |
| 本地数据库 | **Drift（基于 SQLite，启用 FTS5）** | 类型安全、支持迁移、关系查询 + 全文搜索，适合菜谱-记录关系模型 |
| 路由 | **go_router** | 声明式、支持深链接（未来「分享/快捷方式」用得上） |
| 网络 | **dio** | 仅用于检查更新/下载 APK/未来同步 |
| 图片选择 | **image_picker** | 拍照/相册 |
| 图片压缩 | **flutter_image_compress** | 控制存储与备份体积 |
| 文件/路径 | **path_provider / path** | 定位文档目录 |
| 打包/解包 | **archive** | 生成与读取 `.foodlog`(zip) 备份 |
| 校验 | **crypto** | sha256 校验备份/APK 完整性 |
| 版本信息 | **package_info_plus** | 读取当前 versionName/versionCode |
| 安装/打开 | **open_filex** | 触发 APK 安装 |
| 权限 | **permission_handler** | 安装未知来源、存储权限 |
| ID | **uuid** | 实体 UUID 主键 |
| 国际化/格式化 | **intl** | 日期、数字格式化 |
| 轻量配置 | **shared_preferences** | 存设置项（最近备份时间、更新检查开关等） |

> 备选方案：本地库也可用 **Isar**（NoSQL，写入更快），但关系查询与全文搜索不如 Drift 直观；状态管理也可用 Bloc。本文档以 Riverpod + Drift 为准。

## 2. 分层架构

采用 **Feature-first + 清晰分层**，保证可测试、可扩展（未来加同步只影响 data 层）。

```
┌───────────────────────────────────────────────┐
│ Presentation 表现层                              │
│  Screens / Widgets（仅 UI，watch Provider）       │
├───────────────────────────────────────────────┤
│ Application 应用层                                │
│  Controllers / Notifiers（Riverpod，用例编排、状态） │
├───────────────────────────────────────────────┤
│ Domain 领域层                                     │
│  Entities / Value Objects / Repository 接口        │
├───────────────────────────────────────────────┤
│ Data 数据层                                       │
│  Repository 实现 / Drift DAO / 文件存储 / 网络        │
│  └ 未来：BackupService、SyncService 在此扩展        │
└───────────────────────────────────────────────┘
```

依赖方向：表现层 → 应用层 → 领域层；数据层实现领域层接口（依赖倒置）。UI 永远不直接碰 Drift。

## 4. 关键技术决策

### 4.1 本地优先 & 无账号
- 数据全部存本地：结构化数据进 SQLite，图片进应用文档目录 `app_docs/media/`，DB 仅存相对路径。
- 不引入任何登录态；首启直接进首页。
- `app_meta` 表存一个 `install_id`（UUID），用于未来同步去重与备份标识。

### 4.2 UUID 主键 + 软删除 + updated_at
- 所有实体 `id` 用 UUID v4（字符串）。
- 每条记录带 `created_at / updated_at / deleted_at`。
- 目的：让「导入合并」「未来多端同步」可基于 UUID 做幂等 upsert，按 `updated_at` 解决冲突（last-write-wins），软删除可跨端传播删除。

### 4.3 跨平台兼容前置设计
虽然 v1 只发 Android，但为后续多端：
- 平台相关能力（更新安装、文件分享）放在 `services` 并通过接口抽象，Android 提供实现，其它平台可空实现或替代实现。
- 避免使用仅 Android 的私有 API 散落各处。
- 路径、文件操作统一走 `path_provider`，不写死分隔符。

### 4.4 图片存储策略
- 选图后压缩（长边 ≤ 1600px，质量 ~80）再落盘，控制体积。
- 文件名用 UUID，按 `media/yyyy/MM/` 分目录避免单目录文件过多。
- 生成缩略图用于列表（可用内存缓存 + 磁盘缩略图）。
- 备份时图片随 zip 打包；导入时还原到本地并修正路径。

### 4.5 全文搜索
- Drift 启用 SQLite FTS5，对菜谱标题、食材、心得、改良建虚拟表，支持中文需配合分词（v1 可用简单 LIKE + trigram，或集成 `simple` tokenizer / ICU；中文分词复杂，v1 先用 `LIKE %kw%` + 标签筛选，P1 再优化）。

## 6. Android 工程配置要点

- `minSdkVersion`：21（Android 5.0）起步，覆盖面广；目标 `targetSdkVersion` 取当前最新。
- `AndroidManifest.xml` 权限：
  - `INTERNET`（检查/下载更新）
  - `REQUEST_INSTALL_PACKAGES`（安装 APK，Android 8+ 需运行时引导）
  - 相机/相册由 image_picker 处理（Android 13+ 用照片选择器，免存储权限）。
- 配置 `FileProvider` 以便安装下载的 APK（见 06 文档）。
- 应用签名：自建 keystore，**全程使用同一 keystore**，否则自更新覆盖安装会因签名不一致失败。
- `versionCode` 单调递增、`versionName` 用语义化版本（见 06 文档）。

## 7. 质量保障

- **单元测试**：Repository、BackupService（导出→导入往返一致性）、UpdateService 版本比较、合并算法。
- **Widget 测试**：关键页面（记录表单、列表）。
- **静态检查**：`flutter analyze` + `flutter_lints`，CI 强制通过。
- **CI（可选 GitHub Actions）**：分析 + 测试 + 打包 release APK 产物。
