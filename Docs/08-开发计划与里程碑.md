# 08 · 开发计划与里程碑

## 1. 迭代里程碑

> 按「先打通自更新通道 → 再做核心功能 → 后迁移 → 打磨发布」推进，每个里程碑产出可运行版本。
>
> **优先把自更新做在最前面**：自更新通道一旦在真机上跑通，之后每个里程碑的产物都能通过「构建→version.json→自检测→安装」下发到设备，迭代验证不再依赖手动 sideload，显著加快后续节奏。

### M0 · 项目脚手架与架构
- 创建 Flutter 项目，配置包名、最低 SDK、lint。
- 引入核心依赖（Riverpod、Drift、go_router 等）。
- 搭好分层目录结构与主题、路由骨架。
- 配置 keystore 与 release 签名（自更新覆盖安装的前提，**一次性确定并保管好**）。
- **产出**：能跑的空壳 App + 底部导航 4 个空页面。

### M1 · 自更新通道与发布流程（优先打通）
- UpdateService（Android 实现）：检查 version.json、下载、校验 sha256、触发安装。
- AndroidManifest 权限、FileProvider、安装未知应用权限引导、设置项开关。
- 建立发布流程/脚本（构建→sha256→version.json→Release），确定托管位置（GitHub/Gitee/对象存储）。
- 真机走通一次完整链路：发布 v0.0.1 → 发布 v0.0.2 → 旧版检测到 → 下载校验 → 覆盖安装成功。
- **产出**：「骨架版」即具备自检测 + 安装新版能力；此后所有迭代均可经自更新通道下发真机验证。

### M2 · 数据层与核心 CRUD
- Drift 建表（recipes/cooking_logs/tags/recipe_tags/ingredients/media_items/app_meta）。
- 实体、Repository、DAO；预置标签初始化。
- 实现 F-01 想做记录、F-02 做菜记录的创建与列表。
- **产出**：能记录想做的菜、能记录今天做的菜并入库。

### M3 · 菜谱详情与关联、标签、搜索
- F-05 菜谱详情（信息 + 改良汇总 + 做菜时间线）。
- 菜谱状态机、cook_count 维护、想做↔已做流转。
- F-06 标签筛选、F-08 搜索（v1 LIKE 版）。
- **产出**：完整的「收藏→做→沉淀」闭环。

### M4 · 图片与界面打磨
- F-07 图片选取/压缩/落盘/缩略图/大图查看。
- F-12 首页时间线、「今天做什么」推荐、空状态、回收站。
- 按 07 文档完成 Material 3 视觉与交互打磨。
- **产出**：体验完整、好看好用的本地版。

### M5 · 备份与迁移（迁移核心）
- BackupService：导出 `.foodlog`、导入（覆盖/合并）、sha256 校验。
- F-13 自动本地备份 + 备份提醒。
- 往返一致性测试（导出→清库→导入 数据 100% 还原）。
- **产出**：换手机不丢数据，达成关键需求。

### M6 · 测试、加固、v1.0 发布
- 单元/Widget 测试补齐；`flutter analyze` 零告警。
- 真机回归（不同 Android 版本/分辨率）。
- 隐私说明、关于页、帮助（迁移指引）。
- 打 release，走一遍完整自更新验证。
- **产出**：v1.0 正式可用。

### 未来迭代（P2）
- WebDAV 自动同步（复用备份序列化 + 合并算法）。
- iOS / 桌面 / PWA 适配。
- 链接元信息抓取、OCR 菜谱识别、采购清单、数据统计、分享卡片。

## 3. 测试策略

| 类型 | 范围 | 重点 |
| --- | --- | --- |
| 单元测试 | Repository、状态机、版本比较、合并算法 | 业务正确性 |
| 往返测试 | BackupService 导出→导入 | 数据零丢失 |
| Widget 测试 | 记录表单、列表、详情 | 关键交互 |
| 手动/真机 | 多 Android 版本、自更新安装 | 兼容与安装 |
| 静态检查 | flutter analyze + lints | CI 强制 |

**关键验收用例**：
1. 记录想做 → 周末做 → 状态自动转「已做」并出现在时间线。
2. 同一菜谱多次做菜记录，详情页时间线与改良汇总正确。
3. 导出备份 → 卸载重装 → 导入「覆盖恢复」→ 数据与图片 100% 还原。
4. 两设备分别新增/修改 → 合并导入 → 按 updated_at 正确合并、删除正确传播。
5. 发布更高 versionCode → 旧版检测到 → 下载校验通过 → 覆盖安装成功。

## 4. 发布前检查清单（Checklist）

- [ ] `version` 号已递增（versionName + versionCode）。
- [ ] 使用**固定 keystore**签名（与历史版本一致）。
- [ ] release APK 在真机覆盖安装成功。
- [ ] 自更新链路：version.json 已更新、apkUrl 可达、sha256 正确。
- [ ] 备份导出/导入回归通过。
- [ ] `flutter analyze` 无告警，测试通过。
- [ ] 隐私说明声明「数据仅存本地」。
- [ ] keystore、签名密码、托管凭据已安全备份（勿入代码库）。

## 5. 风险与依赖提醒

- **自更新前置的前提是 keystore 先定稿**：M1 发首个版本前必须用 M0 生成的固定 keystore 签名，之后不可更换，否则后续覆盖安装全部失败。
- **keystore 丢失 = 无法继续覆盖更新**，务必多处安全备份。
- 图片多导致备份包大：导出前做压缩与体积提示。
- 中文全文搜索体验：v1 用 LIKE，P1 再上 FTS5 分词。
- 国内网络访问 GitHub 可能慢：自更新源可换 Gitee/对象存储或加镜像。
