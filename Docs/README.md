# 食记 · 做菜日记 APP 开发文档

> 一款「本地优先、无需登录、打开即用」的个人做菜日记 App。
> 帮你随手收藏想做的菜谱，记录每一次下厨的心得与改良。

---

## 一句话定位

把刷视频/图文时看到的好菜谱「先存着，周末做」，把每次做菜的过程、口味、改良「记下来、攒起来」，形成属于自己的私人菜谱库与做菜成长记录。

## 核心设计原则

| 原则 | 说明 |
| --- | --- |
| **本地优先 (Local-first)** | 所有数据默认存在手机本地，离线可用，打开即用，无需联网。 |
| **零账号 (No-Auth)** | 不做注册/登录/用户体系，降低使用门槛。 |
| **数据可迁移** | 通过「备份文件导出 / 导入」保证换手机不丢数据，架构预留未来网盘自动同步能力。 |
| **轻量自更新** | Android 端 App 内自动检测新版本并下载安装，无需上架应用商店。 |
| **跨平台友好** | 采用 Flutter，先发 Android，后续可低成本扩展 iOS / 桌面 / Web。 |

## 两大核心使用场景

1. **灵感收藏（想做清单）**：工作日刷到好菜谱 → 快速粘贴链接/截图/标题 → 标记为「想做」→ 周末翻清单挑菜做。
2. **做菜记录（心得与改良）**：做了菜/学了新菜 → 拍照 + 写心得 + 记录对原菜谱的改良 → 同一道菜可多次记录，沉淀出自己的改良版。

二者通过「菜谱卡」与「做菜记录」两个实体关联：一个菜谱可以拥有多次做菜记录，记录里的改良会回流到菜谱，形成迭代闭环。

## 技术选型速览

| 维度 | 选型 | 备注 |
| --- | --- | --- |
| 框架 | **Flutter (Dart 3, stable)** | 跨平台，先 Android |
| 状态管理 | **Riverpod** | 可测试、解耦 |
| 本地数据库 | **Drift (SQLite + FTS5)** | 关系模型 + 全文搜索 |
| 路由 | **go_router** | 声明式路由 |
| 图片存储 | 应用文档目录 + DB 存相对路径 | 备份时随包打包 |
| 备份格式 | `.foodlog`（zip：JSON + 图片 + manifest） | 见 05 文档 |
| 自动更新 | 自托管 `version.json` + APK | 见 06 文档 |

## 快速开始（开发者）

```bash
# 1. 安装 Flutter SDK (stable)，确认环境
flutter doctor

# 2. 创建项目（建议包名自定义）
flutter create --org com.yourname --project-name foodlog foodlog

# 3. 进入项目，添加核心依赖（详见 02 文档）
cd foodlog
flutter pub add flutter_riverpod drift drift_flutter sqlite3_flutter_libs \
  path_provider path go_router image_picker package_info_plus \
  archive crypto dio uuid intl shared_preferences open_filex permission_handler
flutter pub add --dev drift_dev build_runner

# 4. 运行
flutter run
```

> 完整命名、版本号、构建与发布流程见各分篇文档。
