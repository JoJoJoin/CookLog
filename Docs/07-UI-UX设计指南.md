# 07 · UI/UX 设计指南

## 1. 设计基调

- **温暖、轻松、低门槛**：像一本随手翻的厨房手帐，而非严肃工具。
- **打开即用**：无登录、无引导墙；首屏即内容。
- **快**：记录路径极短，两类核心记录都能两步内完成。
- **图片优先**：做菜是视觉化的，列表与详情以食物照片为主角。

## 2. 信息架构（底部导航）

```
┌──────────────────────────────────────────┐
│                内容区                       │
│                                            │
│                                  [ + ] FAB │
├──────────────────────────────────────────┤
│  首页    │  想做    │  菜谱    │  我的       │
│ (时间线) │ (清单)   │ (库)     │ (设置)      │
└──────────────────────────────────────────┘
```

- **首页**：最近做菜时间线 + 「今天做什么」推荐。
- **想做**：想做清单（灵感收藏）。
- **菜谱**：全部菜谱库（含已做/常做）。
- **我的**：备份/恢复、检查更新、标签管理、回收站、关于。
- **FAB（+）**：点按弹出两个动作 —— 「📝 想做的菜」「🍳 今天做的菜」。

### 3.4 记录表单（想做 / 今天做的）
- 顶部分段或不同入口；字段按 04 文档。
- 大号保存按钮；必填校验最少。
- 「想做」：菜名输入框自动聚焦 + 「粘贴链接」按钮。
- 「今天做的」：顶部加照片区（点击拍照/选图），其后心得、改良、评分、日期。

## 4. 视觉规范

- **设计体系**：Material 3（`useMaterial3: true`），支持浅色/深色，跟随系统。
- **配色**（种子色生成 ColorScheme）：
  - 主色种子：番茄红 `#E8533F` 或 姜黄 `#E8A33D`。
  - 背景：米白 `#FFFBF5`（浅色）/ 深炭 `#1A1715`（深色）。
  - 强调用于 FAB、评分星、状态标签。
- **圆角**：卡片 16dp，图片 12dp，按钮 12dp，整体柔和。
- **字体**：系统默认（中文思源/系统无衬线）；标题加粗，正文常规，行高舒适。
- **图标**：Material Symbols；情绪化点缀可用少量 emoji（🍳📝🎲）。
- **间距**：8dp 基准网格；卡片内边距 16dp。

## 5. 关键交互细节

- **空状态**都给引导文案 + 一键创建按钮（避免冷启动迷茫）。
- **快速记录**：FAB 二级动作用底部弹窗（bottom sheet），大按钮、图标清晰。
- **图片**：列表用缩略图、详情支持多图横滑与捏合缩放。
- **撤销**：删除后顶部 SnackBar 提供「撤销」。
- **评分**：5 星点选，半星可选（P1）。
- **加载/进度**：备份导出、APK 下载显示进度条。
- **无障碍**：保证对比度，控件 ≥48dp 可点区域，重要图标带语义标签。

## 6. 文案语气

- 亲切、口语化、鼓励：
  - 空想做清单：「刷到好菜谱？点 ➕ 存下来，周末就有的做啦」
  - 首条记录：「记录第一道菜，开启你的做菜日记 🍳」
  - 备份提醒：「好久没备份啦，导出一份存到网盘更安心」

## 7. 动效（克制）

- 列表项进入轻微淡入；FAB 展开二级动作有缩放动效。
- 状态变更（想做→已做）给一个小庆祝反馈（如轻微缩放 + 颜色高亮）。
- 避免过度动画影响「快」的核心体验。
